UE5プチコン「とう」

「とう」をテーマに制作した、暗闇×距離の横スクロールランゲームです。 ゴールはなく、どこまで進めたか（距離）をスコアとして競う構成で、 スピード上昇（ギア）によって難易度が上がる設計になっています。

暗闇の中をライトを頼りに進む、横スクロールランゲーム（画像は仮でもOK）

作品概要

本作は「とう」というテーマ案から派生して制作した、暗闇の中を進む横スクロールのランゲームです。 フラッピーバードの下半分のような緊張感をベースにしつつ、ジャンプと横移動で障害を越えながら距離を伸ばしていきます。

ゴール地点は設定せず、どこまで進めたか（到達距離）をスコア化することで、短時間でも何度も挑戦しやすいゲーム体験を目指しました。 ライト演出やスピードギアの要素により、雰囲気と難易度の両方を調整しています。

実装済み機能（要点）

1. プレイヤー操作 / アニメーション

  • 横移動（A/D・←/→）とジャンプ（Space）
  • 移動方向に応じた向き変更（Yaw 0 / 180）
  • Idle / Run / Jump アニメーション導入済み
  • 空中状態で不自然に走らないように調整済み

2. 距離スコア / HUD / リザルト

  • BestX = max(BestX, PlayerX) による最高到達X更新
  • km表記でスコア表示（Float→Text変換の問題を修正済み）
  • HUDで距離表示・Gear表示に対応
  • GameOver時に YourScore / HighScore を表示
  • HighScore はランをまたいで保持

3. ステージ自動生成 / チャンク規格化

  • +X方向へチャンクを自動生成
  • Timer方式（0.7秒運用）で生成チェック
  • 初期先読み生成あり / 先読み距離調整で改善
  • ChunkWidth=1200, TileWidth=200 の統一規格を採用
  • チャンク同士が綺麗に繋がる構成を確認済み

4. Speed（ギア）システム / アイテムスポーン

  • SpeedLevel 上限5、SpeedStep=80
  • ApplySpeedLevel 関数で MaxWalkSpeed を更新
  • Generator側でアイテムを確率スポーン（チャンク埋め込みなし）
  • 連続出現しすぎないように制御設計を導入
  • 速度上昇で難化、そのぶん報酬（倍率）を上げるバランス方針

設計で工夫した点

チャンクの共通規格を決めて接続ズレを防止

ステージ自動生成で破綻しやすい「チャンク接続位置のズレ」を防ぐために、 1チャンクの幅・タイル幅・タイル中心座標・NextAttach位置を共通規格として固定しました。 これにより、新しいチャンクを追加するときもルールに沿って作成しやすくなりました。

GameOver時のスコア更新停止ガード

ランゲームではGameOver演出中にスコアが伸びてしまうと不自然になるため、 bGameOver フラグで BestX の更新を止めるガードを入れました。 これにより、リザルト表示とスコア計測の整合性を保っています。

生成器側にアイテムスポーンを集約

アイテムをチャンクへ直接埋め込まず、Generator側で確率スポーンする構成にすることで、 チャンクの使い回しやバランス調整（出現率・連続制御）がしやすくなるように設計しました。

振り返り・課題

テーマに沿ったシンプルな開発ができ、ライトの調整やアニメーション実装を通してUE5への理解が深まったと感じています。 一方で、実装機能数という意味ではまだ伸ばせる余地があり、構成を保ちながら要素を増やす設計力が次の課題だと考えています。

  • 機能追加（演出・バリエーション）とシンプルさのバランス調整
  • チャンク種類の比率調整（重み付け）による難易度カーブの改善
  • スコア倍率の見せ方や報酬設計のブラッシュアップ
  • ライト演出・UIの最終調整による世界観の強化
